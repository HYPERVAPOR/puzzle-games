# IP用户识别系统

## 功能概述

基于IP地址的智能用户识别系统，实现跨会话的用户身份保持。

## 工作流程

### 1. 首次访问
**输入：** 用户名 + 口令
- 系统验证口令
- 创建新用户ID
- 绑定IP地址 → 用户ID
- **结果：** 注册成功，下次无需口令

**示例：**
```
IP: 192.168.1.100
用户名: 张三
口令: letmein
→ 创建用户 user-xxx，绑定IP
```

### 2. 再次访问（同一IP）
**输入：** 用户名（可选改名）
- 系统识别IP已注册
- 无需验证口令
- 如果用户名不同，更新用户名
- **结果：** 自动登录，保持用户ID不变

**示例：**
```
IP: 192.168.1.100 (已注册)
用户名: 李四（改名）
→ 用户ID保持不变，用户名更新为"李四"
```

### 3. 不同IP访问
- 系统识别IP未注册
- 需要输入用户名 + 口令
- 创建新用户并绑定新IP

## 数据存储

### 文件结构
```
/data/
├── ip-mapping.json    # IP → 用户 映射
└── users.json          # 用户数据（预留）
```

### IP映射格式
```json
{
  "192.168.1.100": {
    "userId": "user-123",
    "username": "张三",
    "lastSeen": "2026-03-11T01:00:00.000Z"
  }
}
```

## 隐私保护

1. **IP地址隐藏** - 显示时隐藏最后一段
   ```
   实际IP: 192.168.1.100
   显示:  192.168.1.***
   ```

2. **数据持久化** - 仅存储必要信息
   - IP地址
   - 用户ID
   - 用户名
   - 最后活跃时间

3. **不存储敏感信息**
   - 不存储口令
   - 不存储浏览器指纹
   - 不存储行为数据

## 使用场景

### 场景1：家中固定IP
```
第1天: 首次访问，输入"张三" + "letmein" → 注册
第2天: 再次访问，直接识别为"张三" → 自动登录
第3天: 改名"李四" → 用户ID不变，用户名更新
```

### 场景2：移动网络（IP变化）
```
第1天: IP 1.2.3.4, 用户"张三" → 注册
第2天: IP 1.2.3.5 (变化) → 需要重新注册
```

### 场景3：办公室共享IP
```
用户A: 注册 "张三"
用户B: 同一IP，改名 "李四" → 更新用户名为"李四"
```

## 技术实现

### 获取客户端IP
按优先级检查以下请求头：
1. `x-forwarded-for` - 代理/负载均衡器
2. `x-real-ip` - Nginx等
3. `cf-connecting-ip` - Cloudflare
4. 降级到 `127.0.0.1`

### API端点

#### GET /api/auth/identify
检查IP注册状态
```json
// 响应
{
  "success": true,
  "data": {
    "registered": true,
    "username": "张三",
    "ip": "192.168.1.***"
  }
}
```

#### POST /api/auth/identify
识别或注册用户
```json
// 请求（新用户）
{
  "username": "张三",
  "passcode": "letmein"
}

// 响应
{
  "success": true,
  "data": {
    "user": {
      "id": "user-123",
      "username": "张三",
      "ip": "192.168.1.***",
      "isNewUser": true
    }
  }
}

// 请求（已注册用户，改名）
{
  "username": "李四"
  // 无需 passcode
}

// 响应
{
  "success": true,
  "data": {
    "user": {
      "id": "user-123",  // ID保持不变
      "username": "李四",
      "ip": "192.168.1.***",
      "isNewUser": false
    }
  }
}
```

## 前端集成

### 页面加载时
```javascript
// 1. 检查IP状态
GET /api/auth/identify
→ 如果已注册：显示"欢迎回来"，隐藏口令输入
→ 如果未注册：显示"首次访问"，显示口令输入
```

### 用户操作时
```javascript
// 2. 加入/创建房间
POST /api/auth/identify
{
  username: "...",
  passcode: isRegistered ? undefined : "..."  // 已注册无需口令
}
→ 获取用户信息
→ 继续房间流程
```

## 优势

1. **用户体验** - 已注册用户无需输入口令
2. **跨会话保持** - 退出重进仍能识别
3. **改名友好** - 同一IP改名更新用户名
4. **隐私保护** - IP地址部分隐藏
5. **简单高效** - 基于IP，无需额外验证

## 局限性

1. **IP变化** - 移动网络、DHCP会导致IP变化，需要重新注册
2. **共享IP** - 多用户共享公网IP时可能互相影响
3. **NAT环境** - 局域网多个用户对外显示同一IP
